# pdf-element-disabler（穴埋め解除ビューア）

先生が穴埋め問題を作るために、答えの上に **白い長方形** を乗せたり、答えを **白文字（背景と同じ色で見えない文字）** にして隠しているPDFを、ブラウザ内で元に戻して答えを表示する静的Webツールです。

PDFはどこにもアップロードされず、すべての処理はブラウザ内で完結します。

デモ: GitHub Pages を有効化すると `https://kain18915.github.io/pdf-element-disabler/` で公開できます。

## 使い方

1. `index.html` をブラウザで開くか、GitHub Pages の URL にアクセスします。
2. `PDFを選択` からローカルPDFを開くか、リポジトリルートに `main.pdf` を置いたうえで `サンプルPDFを開く` を使います。
3. **白い被せ物（長方形）を消す**
   - `白い被せ物をすべて消す` をオンにすると、答えを覆っている白い長方形がすべて消え、下の文字が見えます。
   - 1つずつ消したいときは、PDF上の白い長方形にカーソルを重ねて枠が出た所をクリックします。もう一度クリックすると元に戻ります。
4. **白文字を見えるようにする**
   - `白文字に色を付ける` をオンにすると、白い（見えない）文字に色が付いて読めるようになります。
   - `表示する色` で好きな色に変更できます。
5. うまく消えない / 色が付かないときは、`感度（白とみなす明るさ）` を下げて、薄いグレーなども対象にしてください。
6. 必要に応じて編集済みPDFを保存します。
   - **画像で保存** — 画面上の見た目どおりに、各ページを画像として再構成したPDFを保存します。
   - **ベクターで保存** — 元のPDFのテキスト・ベクター構造を保ったまま、白い被せ物の除去と白文字の色変更を反映します（ファイル名は `*_編集済み_ベクター.pdf`）。

## 仕組み

PDF.js はPDFの内容をすべて 2D canvas に描画します。本ツールはその描画処理に割り込み、

- **白に近い塗りつぶし**（答えを隠す被せ物）→ 描画をスキップして下の文字を見えるようにする
- **白に近い文字**（白文字）→ 指定した色に変えて描画する

ことで、隠された答えを復元します。

## GitHub Pagesで公開する

ビルドは不要です。リポジトリの **Settings → Pages** で、Source を **Deploy from a branch**、Branch を `main`、フォルダを `/ (root)` に設定してください。

PDF.js、PDF.js の CMap / 標準フォントデータ、pdf-lib は CDN から読み込みます。オフライン配信したい場合は各ライブラリと CMap / 標準フォントデータを同梱し、`app.js` の参照先をローカルに変更してください。

## 書き出し方式

| 方式 | 内容 | 長所 | 短所 |
|------|------|------|------|
| 画像で保存 | プレビューと同じ見た目を PNG 画像として各ページに埋め込み | 画面上の表示と一致しやすい | テキスト選択・検索不可、ファイルサイズが大きくなりやすい |
| ベクターで保存 | 元PDFのコンテンツストリームを解析し、白い塗りつぶしの除去と文字色の変更を適用 | テキスト・ベクターを保持、ファイルサイズが小さめ | PDFの内部表現に依存するため、一部のPDFでは反映されないことがある |

ベクター書き出しは `pdf-content-transform.js` が PDF のコンテンツストリーム（塗りつぶし・文字色の演算子）を直接書き換えます。プレビューで検出した被せ物は PDF 座標のバウンディングボックスキーで対応付けられ、個別に消した被せ物だけがベクター書き出しでも除去されます。

## 制限事項

- 答えが **画像**（白い画像や図形画像）で隠されている場合や、透明グループを多用した複雑なPDFでは、被せ物を検出できないことがあります。
- 白文字の復元は、フォントとして描画された文字が対象です。アウトライン化された（図形になった）文字には効かない場合があります。
- **画像で保存** したPDFは、各ページが画像として再構成されます。元PDFのテキストやベクター情報は保持されません。
- **ベクターで保存** は次の場合に限界があります。
  - パターン塗りつぶし、シェーディング、インライン画像による被せ物
  - 非標準カラースペースや複雑な Form XObject のネスト
  - プレビュー（canvas）とコンテンツストリームで座標がわずかにずれるPDFでは、個別クリックで消した被せ物がベクター書き出しで一致しないことがあります
  - ブラウザが `CompressionStream` / `DecompressionStream` に未対応の環境では、圧縮ストリームの再書き込みに失敗することがあります
